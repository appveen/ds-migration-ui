Overview
  • Configuration of Authentication is Changed.
  • ODP had three types of authentication system Local, AzureAD and LDAP, But only one can be used at a time.
  • Data Stack supports three types of authentication system Local, AzureAD and LDAP, and all can be used at the same time.
ODP Authentication Configuration
Configuration
  • Was done using UI, by filling form values.
Pros
  • UI Based Authentication Configuration was User Friendly.
  • Users are imported from the Authentication System using UI.
Cons
  • Only one Authentication System can be used at a time.
  • On Changing Authentication System all Users are deleted.
  • Only Admin Users can import users from Authentication System.
  • Created a lot of confusion with roles and permissions.
Data Stack Authentication Configuration
Configuration
  • Is done using Environment Variables.
  • AZURE_AD_CLIENT_ID
  • AZURE_AD_CLIENT_SECRET
  • AZURE_AD_TENANT
  • AZURE_AD_USER_ATTRIBUTE
  • LDAP_SERVER_URL
  • LDAP_BIND_DN
  • LDAP_BIND_PASSWORD
  • LDAP_BASE_FILTER
  • LDAP_BASE_DN
Pros
  • Multiple Authentication Systems are supported at a time.
  • Any user that has access to all users in the Authentication System can import users.
  • Users are not deleted unless you want to.
  • Users of any Authentication System can be imported using a CSV file.
Cons
  • Configuring any new Authentication System needs a DevOps Personal to Configure at Infra Level.
