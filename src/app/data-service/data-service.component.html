Date Type Changed

        
In data.stack, Date and Date-time fields will cease to exist as a simple key-value pair. Instead, the field
            will
            be saved as an object in the backend.

            The following are the new set of attributes.


        

            
                

                    Field
                    Type
                    Description
                

            
            
                

                    rawData
                    String
                    Raw data entered by the user.
                

                

                    tzData
                    String
                    Date-time form with time zone
                

                

                    tzInfo
                    String
                    Name of the timezone
                

                

                    utc
                    String
                    A string representing the given date in the ISO 8601 format according to universal time.
                

                

                    unix
                    Number
                    Number of milliseconds since 1 January, 1970, 00:00:00 UTC, with leap seconds ignored.
                

            
        


        
E.g., for a field like executionTime, the data will be saved as

        
{{code['date-data'] | json}}

        
If you have existing date and time fields, they would have to be migrated to the new format, data and logs,
            to be
            able to work with data.stack.
